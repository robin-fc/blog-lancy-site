# blog-lancy-site（公众号风格优化）产品设计文档 v2

> **定位：** AI 驱动的新一代公众号内容创作与排版平台
> **核心差异化：** 不做"又一个编辑器"，做**风格化内容工厂**

---

## 一、产品定位

### 1.1 一句话描述

**AI 公众号风格优化平台 — 从文字到爆款，一站式搞定标题、配图、排版、风格。**

### 1.2 核心问题（为什么需要这个产品）

公众号运营者的日常痛点：

| 痛点 | 现状 | 影响 |
|------|------|------|
| 排版耗时 | 手动调格式、选样式、对齐，一篇 30~60 分钟 | 效率极低 |
| 标题平庸 | 想不出好标题，打开率低 | 流量浪费 |
| 配图困难 | 找图→改尺寸→去水印→版权风险，每张图 5~10 分钟 | 质量参差 |
| 风格不统一 | 每篇文章排版风格不同，品牌感弱 | 专业度不足 |
| 多平台分发 | 排好版的文章无法复用到其他平台 | 重复劳动 |
| 数据反馈滞后 | 发完才知道效果，无法预判优化 | 试错成本高 |

### 1.3 目标用户

| 用户类型 | 典型画像 | 核心需求 |
|---------|---------|---------|
| 个人自媒体 | 自由撰稿人/知识博主/KOL | 快速出稿 + 提升阅读体验 |
| 企业新媒体运营 | 品牌号/产品运营/市场部 | 品牌一致性 + 高效批量生产 |
| 内容团队 | MCN/代运营公司 | 协作 + 模板管理 + SOP 标准化 |
| 小白用户 | 刚开始写公众号的人 | 零门槛上手 + 出好看的结果 |

### 1.4 产品边界

**做什么：**
- ✅ 公众号文章的 AI 创作、排版、配图、风格管理
- ✅ 多平台适配输出（公众号/知乎/小红书/头条等）
- ✅ 团队协作与模板资产管理

**不做什么：**
- ❌ 不做全平台 CMS（不是内容管理系统）
- ❌ 不做专业简历编辑与导出（那是 resume-lancy-site 的领域）
- ❌ 不做社群/粉丝管理工具
- ❌ 不做广告投放/数据分析后台

## 三、功能模块详解

### 3.1 模块一：智能编辑器工作台（核心）

这是产品的主体，用户 90% 的时间花在这里。

#### 3.1.1 页面布局

```
┌─ 顶部工具栏 ───────────────────────────────────────────┐
│  [Logo]  [撤销] [重做] | [AI助手▼] [模板▼] [风格▼] | [预览] [发布] │
├──────────┬───────────────────────────┬─────────────────┤
│          │                           │                 │
│  工具面板  │      编辑画布区            │    属性面板     │
│ (可折叠)   │   (所见即所得编辑)         │  (选中元素属性)  │
│          │                           │                 │
│ • 样式库  │                           │  • 字体/字号     │
│ • 素材库  │   [文章内容实时渲染]        │  • 颜色/间距     │
│ • 组件库  │                           │  • 动效设置      │
│ • AI工具  │                           │  • 对齐方式      │
│          │                           │                 │
├──────────┴───────────────────────────┴─────────────────┤
│  底部状态栏：字数 ×××× | 预计阅读 × 分钟 | 自动保存 ✓ | 版本 v3 │
└────────────────────────────────────────────────────────┘
```

#### 3.1.2 四种创作模式

**模式 A：从零写作（AI 辅助）**
1. 用户输入主题/关键词/大纲
2. AI 生成初稿（支持续写/扩写/缩写）
3. 用户在编辑器中修改
4. AI 同时建议：标题备选、段落拆分建议、配图推荐

**模式 B：导入已有文章**
1. 粘贴文本 / 上传 Word / 导入 URL / 导入 Markdown
2. AI 自动识别结构（标题/正文/引用/列表）
3. 一键应用排版模板
4. AI 润色 + 标题优化 + 配图建议

**模式 C：套用模板**
1. 从模板库选择行业/场景模板
2. 替换占位文字和图片
3. 微调样式细节
4. 导出发布

**模式 D：风格迁移（差异化杀手功能）**
1. 输入任意文章链接或粘贴内容
2. 选择目标"风格"（如"得到式"/"夜听式"/"GQ式"）
3. AI 重构全文：
   - 标题重写为对应风格
   - 段落节奏调整（短句密度、换行频率）
   - 金句提炼与加粗强调
   - 引用框/卡片/分割线的自动插入
   - 配图风格匹配

#### 3.1.3 编辑器核心能力

| 能力 | 说明 | 技术实现 |
|------|------|---------|
| 富文本编辑 | 完整富文本，支持所有公众号支持的 HTML 标签 | TipTap / Slate.js |
| 实时预览 | 左侧编辑、右侧手机模拟器实时渲染 | iframe sandbox |
| AI 内联助手 | 选中文字 → 浮动工具栏 → AI 操作 | LLM API + SSE 流式 |
| 拖拽组件 | 卡片/图片/表格/引用/分割线等拖入即用 | dnd-kit |
| 多级撤销 | 支持 50 步撤销/重作 | 自定义历史栈 |
| 自动保存 | 每 30 秒自动保存 + 冲突检测 | IndexedDB + 后端同步 |

### 3.2 模块二：AI 标题引擎

**这是最高频使用的独立功能，可以单独作为入口页。**

#### 3.2.1 功能流程

```
输入文章正文/摘要 → AI 分析内容 → 生成 N 个标题候选 → 用户选择/A/B测试
```

#### 3.2.2 标题生成策略（6 大维度）

| 维度 | 示例 | 适用场景 |
|------|------|---------|
| **痛点型** | "为什么你写的公众号没人看？这 3 个错误 90% 的人都犯了" | 干货类 |
| **数字型** | "7 个技巧让你的阅读量翻倍" | 方法论类 |
| **悬念型** | "我删了这篇文章 5 次，第 6 次终于发出去了" | 故事类 |
| **反常识型** | "别再写长文了，短文才是公众号的未来" | 观点类 |
| **热点型** | "从 xxx 事件看公众号运营的 4 个真相" | 热点追踪 |
| **情感型** | "写给每一个深夜还在写公众号的你" | 情感共鸣 |

#### 3.2.3 标题评分系统

每个生成的标题附带评分：

- **点击率预测分**（0~100）：基于历史数据训练的预估模型
- **违规风险检测**：自动标记可能触发微信审核的关键词
- **情绪标签**：正面/负面/中性 + 强度等级
- **A/B 对比**：同时展示 2 个标题让用户选择偏好

### 3.3 模块三：AI 配图引擎

#### 3.3.1 配图来源（三级优先）

```
用户上传 > AI 生成 > 素材库匹配
```

1. **AI 生图**（主力）：
   - 根据段落内容自动生成匹配插图
   - 支持风格统一（同一篇文章所有图片保持一致画风）
   - 尺寸自动适配（封面图 900×383 / 正文图 1080px宽 / 小图自适应）

2. **智能素材匹配**：
   - 基于语义搜索匹配免费商用素材库（Unsplash/Pexels/自建库）
   - 版权检测：自动标注 CC0 / 商用需授权 / 有风险

3. **用户自定义**：
   - 本地上传 / URL 导入
   - AI 自动裁剪+压缩到最佳尺寸
   - 智能水印添加

#### 3.3.2 封面图生成器

专门针对公众号封面的 AI 工具：

- 输入标题 → 自动生成多款封面设计
- 模板：纯文字大字报 / 图文混排 / 渐变背景 / 人物+文字
- A/B 测试：一次生成 4~8 张供选择
- 尺寸：2.35:1（标准封面）+ 1:1（方形分享图）

### 3.4 模块四：AI 排版引擎

#### 3.4.1 一键排版流程

```
原始文本 → AI 结构分析 → 匹配最优模板 → 应用样式 → 微调建议
```

#### 3.4.2 排版规则引擎（内置专家知识）

| 规则类别 | 具体规则 |
|---------|---------|
| **段落** | 段首不空两格、段间距 1.5em、单段不超过 4 行 |
| **标题** | H2/H3 层级清晰、标题前后留白、重点标题加底色 |
| **强调** | 关键句加粗/变色/引用框，高亮不超过正文的 15% |
| **图片** | 图文间距 1em、图片宽度 100%、自动加注脚 |
| **颜色** | 全文主色 ≤ 3 种、对比度 ≥ 4.5:1（无障碍标准） |
| **字体** | 正文 14~16px、行高 1.75~2、中英文字体分别设置 |
| **引导** | 文末固定区域：关注引导 / 互动提问 / 相关推荐 |

#### 3.4.3 排版风格预设（开箱即用）

| 风格名 | 特征 | 适合场景 |
|--------|------|---------|
| **极简黑白** | 大量留白、黑色正文、仅用灰度 | 科技/商业/个人品牌 |
| **杂志风** | 衬线标题、精致分隔线、引用卡片 | 文化/时尚/深度 |
| **活泼多彩** | 圆角卡片、Emoji点缀、渐变按钮 | 生活/亲子/美食 |
| **学术严谨** | 注释上标、参考文献格式、专业术语高亮 | 教育/科研/医疗 |
| **夜读情感** | 深色背景、暖色文字、短句分行 | 情感/故事/晚安文 |
| **数据报告** | 图表配色、关键数字放大、信息图表化 | 行业分析/财报解读 |

### 3.5 模块五：风格管理中心（差异化核心）

**这是区别于所有现有竞品的核心功能 — 把"排版"升级为"风格资产"。**

#### 3.5.1 品牌风格包

一个完整的品牌风格包包含：

```yaml
brand_style:
  name: "我的品牌"
  colors:
    primary: "#2563EB"      # 主色
    secondary: "#7C3AED"    # 辅色
    accent: "#F59E0B"       # 强调色
    background: "#FAFAFA"   # 背景
    text: "#1F2937"         # 正文色
  typography:
    heading_font: "思源宋体"
    body_font: "系统默认"
    heading_size: ["24px", "20px", "18px"]
    body_size: "16px"
    line_height: 1.8
  components:
    quote_style: "left-border"    # 引用样式
    card_style: "rounded-shadow"   # 卡片样式
    divider_style: "wave"         # 分割线样式
    button_style: "pill"          # 按钮样式
  spacing:
    paragraph_gap: "1.5em"
    section_gap: "2.5em"
  image_treatment:
    border_radius: "8px"
    shadow: "subtle"
    caption: true
```

#### 3.5.2 风格克隆

用户输入一篇喜欢的公众号文章 URL → AI 分析其视觉风格 → 生成对应的风格配置文件 → 可直接应用到自己的文章。

#### 3.5.3 团队风格共享

- 组织管理员创建官方风格包
- 所有成员强制使用（或可选使用）
- 风格版本管理（v1/v2/v3）
- 变更日志与回滚

## 七、竞品分析与差异化策略

### 7.1 主要竞品

| 竞品 | 优势 | 劣势 | 我们的机会 |
|------|------|------|-----------|
| **135编辑器** | 12 年积累、2000 万用户、品牌认知度高 | UI 陈旧、AI 能力弱、移动端体验差 | 更现代的 UX + 更强的 AI |
| **小墨鹰编辑器** | 25 万素材库、AI 快排准确率 98%、500 万用户 | 功能臃肿、学习曲线陡峭 | 聚焦"风格"而非"素材堆砌" |
| **有一云 AI** | 全链条 AI、360+ 模板、热点挖掘 | 排版精细度不够、偏营销向 | 深耕排版品质 + 品牌资产管理 |
| **快编 AI** | 免费、上手快、人性化写作 | 功能较轻、模板较少 | 更专业的企业级功能 |
| **96 微信编辑器** | AI 文章续写/改写/扩充、96 套模板 | 排版自由度低、UI 一般 | 开放式编辑器 + 风格系统 |

### 7.2 差异化矩阵

| 维度 | 竞品普遍水平 | blog-lancy-site |
|------|------------|-----------------|
| **核心理念** | "帮你排版" | **"管理你的内容风格"** |
| **AI 角色** | 辅助工具 | **创作伙伴（内联助手 + 主动建议）** |
| **风格管理** | 无/简单模板切换 | **完整品牌风格包 + 风格克隆 + 版本管理** |
| **标题能力** | 基础生成 | **6 维度策略 + 评分 + 违规检测 + A/B** |
| **配图** | 素材库搜索 | **AI 生图为主 + 版权检测 + 风格统一** |
| **团队协作** | 弱/无 | **风格强制 + 权限管理 + SOP 模板** |
| **多平台** | 仅公众号 | **一键输出 7+ 平台格式** |
| **开放性** | 闭源 SaaS | **核心模板/风格可导出 + API** |

### 7.3 一句话差异化

> **竞品在做"编辑器"，我们在做"内容风格操作系统"。**
> 
> 不是帮你把文章变好看，而是帮你的每一篇文章都长着同一张脸——你的品牌脸。

## 八、MVP 开发优先级

| Phase | 内容 | 核心交付 | 周期 |
|-------|------|---------|------|
| **P0** | 编辑器基础 + 一键排版 | 富文本编辑器、5 个基础模板、AI 排版（结构分析+样式应用）、公众号 HTML 导出 | 2 周 |
| **P1** | AI 标题引擎 | 标题生成（6 维度）、评分系统、违规检测、独立入口页 | 1 周 |
| **P2** | AI 配图 + 封面图 | AI 生图集成、封面图生成器、素材库接入、版权标注 | 1.5 周 |
| **P3** | 风格中心 | 风格包创建/编辑/应用、风格克隆（URL 分析）、3 个预设风格 | 1.5 周 |
| **P4** | 用户系统 + 付费 | 注册登录、Freemium 套餐、积分/订阅、支付对接 | 1 周 |

**总计 MVP 约 9 周。**

## 九、成功指标（North Star Metric）

### 9.1 核心指标

| 指标 | 定义 | 目标（6 个月） |
|------|------|--------------|
| **月活跃文章数** | 每月通过平台完成排版的文章数量 | 10,000 篇 |
| **文章完成率** | 开始编辑 → 完成导出的比例 | > 60% |
| **AI 功能使用率** | 使用过至少一种 AI 功能的用户占比 | > 70% |
| **付费转化率** | 免费用户 → 付费用户的比例 | > 8% |
| **月留存率** | 当月活跃 → 次月仍活跃的比例 | > 40% |

### 9.2 体验指标

| 指标 | 定义 | 目标 |
|------|------|------|
| 从输入到成稿时间 | 用户从开始到导出的平均时长 | < 10 分钟 |
| 一键排版满意度 | 用户对 AI 排版结果不做修改直接使用的比例 | > 50% |
| 标题采纳率 | 用户采用 AI 推荐标题（含微调）的比例 | > 40% |
